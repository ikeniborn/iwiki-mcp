idseveritysectiontextfix / resolutionverdict
F-001 WARNING Mechanism — batch sweep Внутреннее расхождение порядка. Правило Call site: refresh_artifacts идёт «после re-index, до auto-commit»; последовательность свипа перечисляла refresh (шаг 3) до re-index+commit (шаг 4). Внутри wiki_export_okf refresh оказывался до, а не после re-index — два утверждения расходились. Шаги свипа переупорядочены: шаг 3 = re-index, шаг 4 = регенерация через refresh_artifacts «после re-index» + commit — совпадает с правилом write-path. Противоречие снято. fixed · 2026-07-09
F-002 WARNING Reserved files, exclusion, and guards «Exclusion defined once» в едином shared skip path было в напряжении с Files touched, где перечислялись indexer.py/validate.py/lint.py/retrieval.py, каждый под исключение RESERVED_OKF. validate.py/lint.py — stdlib-only и делают проверку имени, а не directory page-walk, поэтому один shared skip path мог не покрывать все четыре точки. Секция переписана: RESERVED_OKF — одна общая константа; исключение только на этапе перечисления страниц (indexer/retrieval + листинг), lint content map из отфильтрованного набора, validate.py/lint.py НЕ изменяются. Files touched обновлён. Locus однозначен. fixed · 2026-07-09
F-003 INFO Authoring rules, README, versioning Заметка в resources.py, правки README/README.ru и bump версии не имели явного критерия приёмки в Testing/Verification (проверялись только осмотром).
