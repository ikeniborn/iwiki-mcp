Итоговый отчёт — embedding-startup-check

Сервер теперь проверяет реальную пригодность embedding endpoint до запуска MCP stdio, отказывается стартовать при ожидаемой ошибке и выводит безопасную понятную диагностику.

Итог: OK · R1–R8 покрыты · оба результата intent подтверждены · 446 passed · документация и локальная iwiki актуальны.

Исполнительное резюме

До изменения процесс мог успешно открыть stdio transport и обнаружить проблему embeddings лишь при первом зависимом tool call. Теперь main() выполняет Config.load(), одноразовый probe_embedding_endpoint() и только после успеха вызывает mcp.run(). Timeout probe — 10 секунд, retries отсутствуют; обычный embed_texts() сохраняет три transient retries.

Ожидаемые ошибки конфигурации, URL, HTTP, transport, JSON и vector shape превращаются в stderr-диагностику и exit 1 без traceback. Literal API key редактируется во всех выводимых полях. Неожиданные программные ошибки по-прежнему пробрасываются.
